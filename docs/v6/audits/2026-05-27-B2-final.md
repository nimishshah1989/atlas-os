## Root Cause of Coverage Gap

2016 is a warm-up year. The `atlas_stock_metrics_daily` table itself has NULL values for:
- `ema_200_stock` (entire 2016): 200-day EMA requires 200 trading days of history
- `extension_pct` (entire 2016): derived from ema_200_stock
- `ret_12m` (entire 2016): 12-month return requires 252 trading days of history

This was confirmed via:
```sql
SELECT COUNT(*) total, COUNT(ema_200_stock) cnt_ema200, COUNT(ret_12m) cnt_ret12m
FROM atlas.atlas_stock_metrics_daily
WHERE date BETWEEN '2016-04-07' AND '2016-12-31';
-- total: 82614, cnt_ema200: 0, cnt_ret12m: 0
```

The 2016 NULLs are expected data behaviour, not computation bugs. Our sector
aggregation correctly propagates NULLs when no constituent data is available.

`rs_12m` coverage gap extends into early 2017 for the same reason (early 2017
stocks missing 12m of price history prior to April 2016 start date).
