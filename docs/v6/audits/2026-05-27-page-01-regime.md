**Verdict:** major

## Route status

`/regime` returns HTTP 404. There is no `frontend/src/app/regime/` directory.

The closest live equivalent is `https://atlas.jslwealth.in/v6/today` (HTTP 200), which is served by
`frontend/src/app/v6/today/page.tsx` + `TodayClient.tsx`. This route combines "market regime" content
(RegimeIndicator, DiffSinceYesterdayPanel, BookAtAGlance) with signal-calls and the cell matrix —
it is a v6 "Today" dashboard, not a dedicated regime page.

`/v6/regime` also exists (HTTP 200) and is served by a separate route (`frontend/src/app/v6/regime/`).
That page uses `RegimeHero.tsx` which contains the hero stat strip and 12-week journey strip —
the closest match to mockup 01.

Audit below is against `/v6/regime` as the canonical regime page.

## Token compliance

- [x] Tailwind classes: `RegimeHero.tsx` uses `bg-[#2F6B43]`, `bg-[#1D9E75]`, `bg-[#B8860B]`, `bg-[#B0492C]` as inline hex values in `regimeTailwindBg()` — these are hardcoded color overrides, not semantic token classes. The semantic equivalents (`bg-signal-pos`, `bg-teal`, `bg-signal-warn`, `bg-signal-neg`) should be used instead.
- [x] No inline rgba() in RegimeHero itself, but hardcoded hex strings at lines `return 'bg-[#2F6B43]'` etc. violate the "no inline hex" discipline.
- [x] Fonts: Source Serif 4 / Inter / JetBrains Mono confirmed via Tailwind `font-serif`, `font-sans`, `font-mono` classes.

## Component reuse

- [x] `/v6/regime` page imports `RegimeHero` from `@/components/v6/RegimeHero`; page shell is at `frontend/src/app/v6/regime/page.tsx`.
- [x] `/v6/today` uses `RegimeIndicator`, `DiffSinceYesterdayPanel`, `BookAtAGlance`, `RecentSignalCalls`, `SectorLadder`, `CellMatrix` — all from `components/v6/`.
- [ ] The "four signals tiles" section is absent from both pages; no `PulseTiles` component exists in `components/v6/`.
- [ ] The "cells favoured" section (3-col cell cards) is absent; no equivalent component in `components/v6/` for regime-scoped cell cards.

## Data correctness

- [x] RegimeHero renders real regime state, deployment multiplier, days-in-state, and flip probability from `atlas.atlas_regime_states`.
- [ ] Regime context tile shows `—` for typical-length, next-state probabilities: `RegimeHero.tsx` has no query for historical base rates; only current-state fields are shown.
- [ ] Four signal inputs (Small-cap RS Z, Breadth %, India VIX, Dispersion) show no values anywhere on `/v6/regime` — data exists in `atlas.atlas_regime_inputs` but is not fetched.
