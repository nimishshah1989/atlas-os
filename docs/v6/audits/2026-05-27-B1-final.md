## BLOCKED Items

### NSE FII/DII Historical Data

All known NSE historical FII/DII endpoints are 404 or bot-blocked as of 2026-05-27:
- `archives.nseindia.com/content/nsccl/fao_fo_all_foii_dii.csv` — 404 (confirmed from EC2)
- All other archive paths for historical CSV → 404 or redirect to homepage
- NSE website FII data pages → JS challenge, no curl access
- nsetools Python library → uses same dead URLs

Only current-day data is available via the `fiidiiTradeReact` API endpoint.
`fetch_fii_dii_today()` was added for incremental daily collection going forward.
Historical backfill requires a paid data vendor (CMIE, ACE Equity, or similar).

Until historical FII/DII data is sourced, these columns will remain at effectively 0% coverage for the 2016-present range and the B.1 G3 criterion cannot be fully met.

## risk_free_91d Proxy

FRED series INTGSB91D156N (91-day T-bill) returns HTTP 400 — discontinued or unavailable.
Proxy used: IRSTCI01INM156N (RBI call money/interbank overnight rate, monthly).
- Verified: 123 rows 2016-2026-03 on EC2
- Documented in atlas/ingest/macro/fred_ingest.py docstring
- This is a short-term rate proxy, not the exact 91-day T-bill; acceptable for regime computations

## Forward-Fill Architecture

Two-tier SQL correlated subquery UPDATE:

1. Monthly columns (india_10y_yield, risk_free_91d, cpi_yoy): fill up to 31 days forward
2. Trading-day columns (us_10y_yield, brent_inr, vix_9d, cpi_yoy): fill weekends/holidays

Both use the same SQL template validated against an allowlist. All column names are validated before injection into the SQL string.
