Design principles applied in this mockup:
  • One verdict, one row. BUY · ACCUMULATE · WATCH · HOLD · AVOID · SELL — CONTEXT.md canonical vocabulary. Plus new WAIT (proposed addition to CONTEXT.md) for the cell-POS-but-veto-blocked case.
  • One expected return number. Tier label is a colored chip next to the number, not a competing axis.
  • First-called timestamp + since-call return. Every verdict is a trackable signal_call_id; the page shows when it was first issued and what's happened since. Existing schema (atlas_signal_calls), just not surfaced.
  • Why-strip with 4 chips. Cell · Weinstein · Investable · Sector. Each chip is colored pass/warn/fail. Conflict cases (Case 2) get an extra Reconcile chip that names the override.
  • "Show the math" is collapsed. IC values, individual gate breakdown, composite components, σ band, cell-level history — all of this is one click away for the analyst, hidden from the trader by default.
  • Ownership-aware rendering. Same cell math; verb changes (BUY/ACCUMULATE, AVOID/SELL) based on whether the user holds a position. Matches CONTEXT.md lines 520-525.
  • Tracking since first call (Case 1). Predicted band ± σ shown next to realized. Drift detection becomes visible: if realized falls outside the band, the signal_call_id gets flagged in atlas_drift_event_log (already schemaed).
Open questions to resolve before we build this:
  • Add WAIT to CONTEXT.md vocabulary? Currently CONTEXT.md only has POSITIVE/NEUTRAL/NEGATIVE → 6 display labels. There's no canonical label for "positive cell blocked by Weinstein or gate veto." Proposal: add WAIT as a 7th display label, semantically "positive cell but precondition unmet."
  • Weinstein veto precedence. Does Stage 4 alone veto a positive cell (current proposal), or do we need Stage 3 + Stage 4 to veto? Stage 3 (topping) is ambiguous — a positive cell during Stage 3 might be early entry to the next leg, or it might be a trap.
  • How fast can we capture uptrend transitions (Stage 1→2)? Fund manager feedback: Weinstein 30W MA is slow. Possible fix: add a 10W MA fast-confirm overlay; promote Stage 1 to Stage 2 when price > 10W MA AND 10W MA slope > 0 for 3 consecutive weeks, even if 30W slope hasn't turned. This trades a small false-positive rate for faster entry.
  • Drift detection threshold. Realized outside predicted ± 2σ = drift event. What's σ for a 3M call? Calibrated per-cell from backtest, or one universal value? CONTEXT.md line 115 implies per-call (predicted_excess, σ_predicted both stored).
  • Cross-nav still missing. Homepage top-conviction list, peer table, anywhere a symbol appears as text — must Link to /stocks/[symbol]. Separate pass, batched with the verdict rebuild.
