08Calibration loops — staying on goal

You said "constantly calibrating to our goal post". Four explicit loops do this:

Loop 1 — Per-edit calibration (sub-second)

Every edit goes through 8 PreToolUse hooks + 4 PostToolUse hooks. Float for money, bare except, hardcoded creds, etc. = instant block. The system literally cannot drift on engineering rules.

Loop 2 — Per-chunk calibration (every ~hour)

Every chunk loop runs /grill-with-docs against CONTEXT.md before code edit, and /review + /codex review before merge. Terminology + architecture cannot drift across chunks because the grill catches it and outside-voice catches missed grills.

Loop 3 — Periodic calibration (monthly)

/zoom-out cathedral test asks: "Is what we're building still the v6 the CEO plan describes?" If yes, continue. If no, surface the deviation as a planning question. Strategy cannot silently drift past a month.

Loop 4 — Continuous calibration (live, post-Phase-5)

The drift detector compares realized vs predicted excess per cell. Cells that fail the |Z|>2.5 over 10 days threshold auto-deprecate (locked in CONTEXT.md). Methodology cannot silently drift in production — the system retires its own broken cells.
