TradingView × Atlas Integration

Status: draft · pending eng-review  |  Date: 2026-05-28  |  Approach: B — Cached + MCP  |  Depends on: SP08 (Kite auth in place), existing TV_SESSION_ID config  |  Deferred: Pine Script webhook receiver, live Kite execution (v7)

1. Problem Statement

Atlas v6 currently makes conviction calls (POSITIVE / NEUTRAL / NEGATIVE, Tier 1–5, Weinstein stage) but surfaces no fundamental context alongside them. A fund manager looking at a POSITIVE call on HDFC Bank has to leave Atlas, open TradingView, and manually verify P/E, EPS trend, and technical momentum before acting. This context-switch breaks the decision loop Atlas is meant to close.

In parallel, Atlas manages paper portfolios (atlas_paper_portfolio) but has no performance analytics layer — no Sharpe ratio, no benchmark comparison, no sector composition view. A fund manager cannot answer "how is my model portfolio actually doing vs Nifty?" from within Atlas today.

TradingView is the primary charting tool used by fund managers alongside Atlas. The goal is not to replace it — it is to make Atlas the hub that enriches decisions with TV data and tracks portfolio outcomes, while TV remains the charting tool of choice.

2. Solution Overview

One-line summary: Embed TradingView's Advanced Chart on the stock deep-dive page, pull fundamental + technical data from TV's screener nightly into a cached table, build a portfolio analytics layer inside Atlas, generate TV-importable CSV exports, and wire up the community TradingView MCP server for ad-hoc agent queries.

Five capabilities, built as a single atlas/tv/ module:

  1. TV Advanced Chart Widget — embedded right panel on stock deep-dive (Bloomberg-style 36/64 split)
  2. TV Screener cache — nightly batch pull of fundamentals + technical ratings into atlas.tv_metrics
  3. Portfolio analytics — Sharpe, Alpha, Max Drawdown, Time-Weighted Return, sector composition, benchmark vs Nifty 50
  4. TV Portfolio CSV export — downloadable per-portfolio CSV in TV Portfolio import format, with in-app upload guide
  5. Community TV MCP — mikeh-22/tradingview-mcp wired as an Atlas MCP server for Hermes (SP07) and ad-hoc Claude queries

What is explicitly deferred: Pine Script webhook receiver, live Kite execution routing, TV Portfolio API sync (no public API exists today — designed for easy addition when TV ships one).

5.2 GET /v1/portfolios/{id}/analytics — user-auth required

Returns computed performance analytics for a paper portfolio. Benchmark is Nifty 50 (symbol NIFTY 50 in de_index_prices). Price history from de_equity_ohlcv.close_adj. Instrument lookup: atlas_paper_portfolio.instrument_id → atlas_instruments.symbol for TV metrics join and sector rollup.

Response 200:
{
  "data": {
    "portfolio_id": "...",
    "period_days": 365,
    "twr": 0.183,                 // Time-Weighted Return
    "cagr": 0.183,                // same as TWR for periods ≤ 1y; annualised otherwise
    "sharpe": 1.42,
    "sortino": 1.87,
    "calmar": 2.1,
    "alpha": 0.064,               // Jensen's alpha vs Nifty 50 (see formula in pre block above)
    "beta": 0.91,
    "max_drawdown": -0.087,       // peak-to-trough: min(portfolio_value / running_max - 1)
    "alpha": 0.064,               // Jensen's alpha: Rp - [Rf + β(Rm - Rf)]
                                  // Rf = 91-day T-bill (IRSTCI01INM156N proxy), Rm = Nifty 50 TWR
    "benchmark_nifty50_twr": 0.119,  // Nifty 50 TWR same period (primary benchmark)
    "benchmark_tier_blend_twr": 0.104, // cap-weighted blend: Large→Nifty100, Mid→Nifty Midcap 150,
                                       // Small→Nifty Smallcap 250. Weights from portfolio holdings.
    "sector_composition": [
      { "sector": "Banking", "weight": 0.28 },
      { "sector": "IT", "weight": 0.19 }
    ],
    "daily_returns": [...],       // [{date, portfolio_return, benchmark_return}] for chart
    "holdings_count": 14,
    "as_of_date": "2026-05-28"
  },
  "meta": { "data_as_of": "2026-05-28T21:03:14+05:30", "fetched_at": "..." }
}

5.3 GET /v1/portfolios/{id}/export/tv-csv — user-auth required

Streams a CSV download in TradingView Portfolio Management import format. Returns Content-Disposition: attachment; filename="atlas-portfolio-{id}-{date}.csv". Logs export to tv_portfolio_exports.

6. Frontend Changes

6.1 Stock deep-dive — Bloomberg split layout

The stock deep-dive page (/stocks/[symbol]) gains a 36/64 split layout:

  • Left 36% — Atlas verdict badge + conviction pillars (existing) + TV Technical Rating badge (TVMetricsBadge) + fundamentals grid (PE, P/B, ROE, EPS growth, EV/EBITDA, D/E) + portfolio position strip if held.
  • Right 64% — TVChartPanel: TradingView Advanced Chart Widget iframe. Symbol auto-set to NSE:{symbol}. Theme: dark (#131722). Default interval: 1D.

TV badge visual contract — explicitly separate from Atlas verdict: the badge is labelled "TV Consensus:" (not just a raw rating word), uses a blue-tinted palette (#2962FF spectrum, not Atlas teal/green), and sits in its own named row in the left panel under the heading "TradingView Analysis". The Atlas verdict section and the TradingView section are visually separated by a rule. The API field is named tv_recommend_label (not recommend_label) in all code and API responses to prevent ambiguity. Colour mapping: Strong Buy = #1565C0, Buy = #1976D2, Neutral = #78909C, Sell = #E65100, Strong Sell = #B71C1C. Never shares a colour with Atlas's teal (#1D9E75) or green (#2F6B43).

6.2 Portfolio analytics page — /portfolios/[id]/analytics

New page with four zones:

  1. Performance header — TWR, Sharpe, Alpha, Max Drawdown, Calmar as metric tiles. Shows two benchmark labels: "vs Nifty 50" (primary) and "vs tier blend" (secondary, computed from portfolio's actual cap tier weights).
  2. Benchmark chart — Recharts line chart with three lines: portfolio cumulative return, Nifty 50 (solid), tier-blend benchmark (dashed). Period selector: 1M / 3M / 6M / 1Y / All. Drawn from daily_returns array which carries {date, portfolio_return, nifty50_return, tier_blend_return}.
  3. Sector composition — Recharts pie/donut. Click sector → filter holdings table.
  4. Holdings table — AG Grid: symbol, qty, avg cost, current price, unrealised P&L, weight, TV Recommend label. Sortable, CSV export button in toolbar.

6.3 TV CSV export — in-app guide

CSV export button on the portfolio analytics page triggers /api/v1/portfolios/{id}/export/tv-csv. Below the button: a collapsed "How to import into TradingView Portfolio" guide with step-by-step instructions and a screenshot placeholder. This removes the need for a user to figure out the import flow independently.

7. MCP Integration

The community MCP server mikeh-22/tradingview-mcp is added to Atlas's MCP config. It authenticates via TV_SESSION_ID + TV_SESSION_SIGN (already in atlas/config.py).

Tools exposed and their Atlas use cases:

ToolAtlas use case
get_quoteHermes agent (SP07) — real-time price in agent responses
get_ohlcvAd-hoc Claude queries: "show me HDFC Bank last 10 days"
screen_stocksCross-reference TV screener results against Atlas conviction calls
get_newsHermes agent — surface relevant news in stock briefs
get_watchlistSync user's TV watchlist with Atlas universe
get_alertsSurface pending TV price alerts in Atlas dashboard (future)
get_chart_layoutPre-load saved TV chart layouts from within Atlas deep-dive
search_symbolsSymbol resolution for non-standard tickers
Scope: All MCP tools are read-only. No write-back to TradingView. MCP is additive — removing it does not break any core Atlas feature.
Reliability note: mikeh-22/tradingview-mcp is a community server — not maintained by TradingView. If it breaks or goes unmaintained, the fallback is direct tradingview-screener Python lib calls (already used by screener.py for TV-02). The MCP provides richer tools (news, watchlists, alerts) beyond what screener.py covers; those features degrade gracefully to "unavailable" rather than breaking Atlas. Monitor for upstream breakage in monthly ops review.

11. Success Criteria

  • tv_metrics populated for ≥ 95% of NSE symbols after first nightly run
  • Deep-dive page loads with TV chart + fundamentals in < 1.5 s (cached data, no live TV API call on page load)
  • /v1/tv/metrics/{symbol} returns is_stale: true correctly when fetched_at > 26 h
  • Portfolio analytics: Sharpe and Max Drawdown values match hand-computed test fixture to 4 decimal places
  • TV CSV export downloads correctly and passes TV Portfolio manual import test
  • MCP get_quote + get_news tools respond within 2 s from Hermes agent
  • Zero synthetic data — all portfolio P&L computed from real prices_daily rows
