Stock Detail Page Redesign

Atlas OS · Spec 2026-05-29 · Status: APPROVED FOR IMPLEMENTATION

Core principle: The page answers one question — "Is this stock actionable right now, and why?" — not "what are all the signals for this stock."

Reading contract: Verdict in 5 seconds. Story in 30 seconds. Full evidence available without hunting.

Primary user: A portfolio manager or analyst fluent in the Weinstein/RS framework who wants Atlas to lead with synthesis, then trusts them to verify the evidence.

Page Structure — Seven Sections

Single vertical scroll. No tabs. No fixed panels. Section weight matches decision importance.

Section Order

  1. Event Header — What happened, when, and where in the peer ranking
  2. The Chart — TradingView price chart + Atlas commentary + 5 fundamental pills
  3. RS Confirmation — Stock vs Sector ratio + Stock vs Nifty ratio (the timing gate)
  4. Lifecycle Position — How fresh, how much upside captured, volume conviction
  5. Peer Matrix — 4–5 sector peers vs parent stock across 8 metrics
  6. Supporting Detail — Conviction tape, stage history, component scores (for analysts)
  7. Act — Position sizing if portfolio active

Atlas Commentary

  • Commentary is generated from Atlas data, not hardcoded. Rules:
  • Breakout reference: if stock crossed EMA 200 in the last 30 days → mention breakout and price level
  • Volume context: if latest 20D vol > 63D vol avg × 1.5 → "on expanding volume"; else "volume holding"
  • EMA structure: mention EMA 20 slope (rising/flat/falling) and extension from EMA 20
  • Higher low: if state transitioned from Stage 1 → Stage 2 → note the base formation period
  • Commentary composed server-side from metricHistory + stockState + stateHistory

Data Requirements — New API Endpoint

⚠ New data: RS ratio time series requires a new backend endpoint. See Backend Sub-Plan.
  • Endpoint: GET /v1/stocks/{symbol}/rs-ratios?days=252
  • Returns two series:
    • vs_sector: daily ratio of stock close ÷ linked sector index close
    • vs_nifty50: daily ratio of stock close ÷ Nifty 50 close
  • Sector index resolved via atlas_universe_stocks → sector → de_indices.linked_sector → de_index_prices
  • Resistance level = 52-week high of the ratio (rendered as a horizontal dashed line)
  • Status badge computed from: if current ratio > 52W high × 0.97 → BREAKING OUT; if within 3% below → AT RESISTANCE; else → BELOW RESISTANCE

Chart Spec

  • Recharts LineChart — not TradingView (ratio is a derived series, not a tradeable instrument)
  • Ratio line: stroke="#1D9E75", strokeWidth=2, no dots
  • 52W high resistance line: ReferenceLine y={resistance} stroke="#B8860B" strokeDasharray="4 2"
  • X axis: dates, abbreviated (Jan/Feb/...)
  • Y axis: ratio value, 3 decimal places
  • Tooltip: date + ratio value
  • Height: 120px per chart

Commentary

  • Generated server-side from ratio status badges
  • Template: "{sector} confirmed — {detail}. {nifty} — {detail}. {action recommendation}."

Data Sources

  • Dwell days: computed from stateHistory — days since first entry into current state
  • Stage duration context: hardcoded typical ranges per stage (Stage 2A: 10–90 days; Stage 2B: 60–360 days; Stage 2C: 90–400 days)
  • Extension from EMA20: metricHistory[latest].ema20_ratio — already computed as price/EMA20
  • Extension from EMA50: compute as (price - ema50) / ema50 from metricHistory
  • Volume conviction: metricHistory[latest].vol_ratio_63d — 20D avg ÷ 63D avg
  • OBV direction: footerMetrics.obv_slope — positive/flat/negative

Volume Conviction Label

  • vol_ratio > 1.3 → "↑ Expanding" (signal-pos)
  • 0.8 < vol_ratio ≤ 1.3 → "→ Stable" (ink-3)
  • vol_ratio ≤ 0.8 → "↓ Fading" (signal-neg)

Peer Selection

  • Top 4 stocks by market cap in the same NSE sector as the parent stock
  • Exclude stocks in Stage 4 or Uninvestable
  • Parent stock always shown as first row (highlighted), even if not top 4 by mcap
  • Maximum 5 rows total (parent + 4 peers)

Column Definitions

  • Stage: Weinstein state label shortened (2A, 2B, 2C, 1, 3, 4)
  • Conviction: direction (↑/↓/→) + strength label from convictionTape[dominant_tenure]
  • RS vs Nifty: rs_pctile_3m trend shown as % diff from 50th percentile baseline; direction arrow
  • EMA20 Slope: from metricHistory — classify as Rising/Flat/Declining based on 4-week slope
  • EMA50 Slope: same classification on 8-week slope
  • OBV: from footerMetrics.obv_slope — Positive/Neutral/Negative
  • 3M Return: from metricHistory.ret_3m
  • Extension: price / EMA20 − 1, shown as percentage

Color Coding

  • Positive signal → signal-pos text
  • Negative signal → signal-neg text
  • Mixed/warning signal → signal-warn text
  • Neutral → ink-3 text

New API Endpoint Required

  • GET /v1/stocks/{symbol}/peer-matrix
  • Returns parent stock + top-4 peers with all 8 metrics pre-computed
  • Single endpoint to avoid N+1 fetch per peer

Section 6 — Supporting Detail

For the analyst who wants to verify every data point. Present but not prominent — sits below the decision-relevant sections.

Content (in order)

  • Conviction Tape: Four tenure cells (1M / 3M / 6M / 12M) each showing direction + IC score. Compact horizontal strip using existing ConvictionTape component
  • Stage History: 252-day color-coded strip. Compact version — 8px height bars. Existing DwellTimeline component
  • Component Scorecard: Existing ComponentScorecard component. Rendered at full detail for analysts who want rule-level verification
  • Hit Rate: One-liner. Existing HitRateRow component

What is REMOVED from this section vs current page

  • Within-state peers table (30 rows)
  • StockDeepDiveBody (8-chart metric history with time range picker)
  • OBV chart as a standalone section (now in Lifecycle)
  • ATR gauge as a standalone section (now captured in commentary)
  • Standalone TVChartPanel at bottom

Section 7 — Act

Unchanged from current design. Position sizing if portfolioId query param is present. Existing ActButton component.

Backend Extensions Required

This redesign requires two backend additions before frontend work can complete.

Extension A — TV Fundamentals in screener.py BACKEND

  • Add to atlas/tv/screener.py → _COLUMNS: "price_earnings_ttm", "price_sales_current", "price_book_fbs", "debt_to_equity", "return_on_equity"
  • New migration (119) adding 5 nullable NUMERIC columns to atlas.tv_metrics: pe_ttm, ps_current, pb_fbs, debt_to_equity, roe
  • Update _upsert_rows to include new columns
  • Update GET /v1/tv/metrics/{symbol} response to include new fields

Extension B — RS Ratio Endpoint BACKEND

  • New endpoint: GET /v1/stocks/{symbol}/rs-ratios?days=252
  • Queries: de_equity_ohlcv for stock closes + de_index_prices for sector index and Nifty 50 closes
  • Sector index resolved: atlas_universe_stocks.sector → atlas_sector_master.primary_nse_index → de_index_prices
  • Returns: { vs_sector: [{date, ratio}], vs_nifty50: [{date, ratio}], sector_index_code: string }
  • 52W resistance = max ratio over the returned window (computed in endpoint, not frontend)

Extension C — Peer Matrix Endpoint BACKEND

  • New endpoint: GET /v1/stocks/{symbol}/peer-matrix
  • Returns: parent stock + top 4 sector peers (by market cap, excluding Stage 4/uninvestable) with all 8 metrics pre-computed
  • Avoids N+1 fetches on frontend
